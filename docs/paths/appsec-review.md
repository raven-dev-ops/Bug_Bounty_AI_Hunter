# Path: AppSec Review

This path focuses on structured review of an AI-enabled application.

## Steps
1. Assemble the target profile and scope (`TARGET_PROFILE.md`).
2. Generate a threat model and dataflow map (`THREAT_MODEL.md`).
3. Use checklists to review RAG, embeddings, logging, and agent surfaces.
4. Plan tests with safe canaries (`TESTING.md`).
5. Produce an internal report and remediation plan (`REPORTING.md`).

## Helpful references
- `KNOWLEDGE_FORMAT.md` and `KNOWLEDGE_TAGS.md` for knowledge cards.
- `ROE.md` for safety and authorization guardrails.
